# 架构与工作原理

## 目标

微信视频号卡片通常包含：

```text
objectId
objectNonceId
```

Finder 网关将同一张卡片的这两个字段转换为：

```text
https://weixin.qq.com/sph/SHORT_CODE
```

## 默认本地架构

```text
微信机器人或本地程序
-> 127.0.0.1:8790
-> FastAPI
-> Playwright 持久化 Chromium
-> 已登录的视频号管理后台
-> webpack 短链 service
-> sph
```

所有组件默认运行在同一台 Windows 电脑上。Chromium 直接显示在桌面，不需要
虚拟显示器、VNC 或浏览器隧道。

## 组件

### finder-gateway

负责：

- 启动持久化 Chromium。
- 打开视频号管理后台。
- 复用已登录页面中的 webpack 模块。
- 调用短链生成 service。
- 提供 FastAPI 接口。
- 验证 API Key。

核心实现：

```text
finder_gateway/app/main.py
```

### 本地 Chromium

本地模式使用 Playwright 下载的 Chromium，并以可见窗口运行：

```dotenv
FINDER_BROWSER_HEADLESS=false
```

登录数据保存在：

```text
data/chromium-profile
```

该目录包含敏感登录状态，已被 Git 忽略。

### wxsph-api

可选组件，负责把已经生成的 `sph` 解析为视频信息。

当前实现依赖元宝会话，因此：

- 需要有效 Cookie。
- 会话可能过期。
- 与 Finder 短链生成服务互相独立。
- 不启用时不会影响 Finder 网关。

## 数据流程

```text
卡片 XML
-> 调用方提取 objectId + objectNonceId
-> POST /api/v1/finder/share-url
-> FastAPI 验证 API Key
-> Playwright 在管理后台页面执行 JavaScript
-> webpack service.getObjectShortLink()
-> 返回 sph
```

可选后续：

```text
sph
-> /api/wxsph
-> 元宝解析
-> 标题、作者、封面和媒体候选地址
```

## 参数来源

原始卡片 XML 示例：

```xml
<objectId>OBJECT_ID</objectId>
<objectNonceId>OBJECT_NONCE_ID</objectNonceId>
```

调用方应在机器人消息回调或原始消息结构中提取字段。两个值必须来自同一张
卡片。Finder 网关不会根据 object ID 推算 nonce。

## webpack service 定位

管理后台使用：

```javascript
window.webpackChunkfinder_helper_web
```

网关通过临时 chunk 回调捕获 webpack `require`，然后：

1. 优先尝试已知模块 `91847`。
2. 递归查找包含 `getObjectShortLink` 的导出对象。
3. 如果模块 ID 变化，扫描模块源码中的方法名称。
4. 缓存已经定位的 service。

业务调用结构：

```javascript
service.getObjectShortLink({
  exportId: String(objectId),
  nonceId: String(objectNonceId),
  scene: 40
});
```

数值形式的卡片 `objectId` 可以作为 `exportId` 使用。

## 返回值处理

网关兼容管理后台可能返回的字段：

```text
data.shortUrl
data.feedH5Url
data.shareUrl
```

最终 URL 必须匹配：

```text
https://weixin.qq.com/sph/...
```

否则网关返回错误，不把未知 URL 交给调用方。

## 就绪状态

以下状态表示浏览器和短链模块已经就绪：

```json
{
  "browser_started": true,
  "ready": true,
  "reason": ""
}
```

如果 `ready` 为 `false`：

1. 检查桌面 Chromium 是否已经扫码登录。
2. 进入内容管理页面。
3. 打开一次分享菜单。
4. 必要时调用 reload 接口。

## 并发模型

一个 Chromium context 和一个页面由异步锁保护，同一时间只执行一个页面操作。

该模型适合个人机器人、消息回调和小规模调用，不适合匿名高并发服务。增加
Uvicorn worker 数量也不能安全扩展同一个 Chromium profile。

## 本地网络边界

`start_local.ps1` 默认绑定：

```text
127.0.0.1:8790
```

因此只有本机程序可以访问。需要局域网调用时可以显式绑定 `0.0.0.0`，但必须
增加 Windows 防火墙来源限制。

## 可选 Docker/VPS 架构

容器模式在 Linux 中增加：

```text
Xvfb
openbox
x11vnc
noVNC
```

这些组件只用于远程查看 Chromium 和扫码。Docker Compose 默认仍将 Finder API
和 noVNC 绑定在服务器 `127.0.0.1`，公网通过 Nginx 只代理短链生成路径。

noVNC 应通过 SSH 隧道访问，不能直接公开。

## 安全边界

- Finder 网关不读取或返回浏览器 Cookie。
- API 响应不包含 nonce。
- 应用日志只允许记录 nonce 长度。
- Uvicorn access log 默认关闭。
- profile、`.env`、缓存和日志不会进入 Git。
- 本地服务默认不监听公网网卡。
- 远程部署必须增加 HTTPS、限流和密钥撤销机制。
