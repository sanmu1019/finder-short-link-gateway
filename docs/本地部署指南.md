# 本地部署指南

本文以 Windows 本地运行作为默认方式。Finder 网关、Chromium 和调用程序
运行在同一台电脑，不需要 Docker、VPS、noVNC、Nginx 或本地隧道。

## 运行结构

```text
机器人或本地脚本
-> http://127.0.0.1:8790
-> FastAPI Finder 网关
-> 本机 Playwright Chromium
-> 视频号管理后台
-> sph 短链
```

## 前置条件

建议：

```text
Windows 10/11
Python 3.12+
Git
可正常访问视频号管理后台的网络环境
```

检查：

```powershell
python --version
git --version
```

## 获取代码

```powershell
git clone YOUR_REPOSITORY_URL
cd finder-short-link-gateway
```

仓库根目录应包含：

```text
setup_local.ps1
start_local.ps1
.env.example
finder_gateway/
```

## 首次安装

执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\setup_local.ps1
```

脚本会自动：

1. 创建 `.venv` Python 虚拟环境。
2. 安装 `finder_gateway/requirements.txt`。
3. 下载 Playwright Chromium。
4. 首次运行时复制 `.env.example` 为 `.env`。

安装结果：

```text
.venv/
.env
```

这两个路径都不会提交到 Git。

如果下载 Chromium 失败，确认网络后重新执行：

```powershell
.\.venv\Scripts\python.exe -m playwright install chromium
```

## 配置

编辑 `.env`：

```dotenv
FINDER_API_KEY=change_me_to_a_long_random_key
FINDER_PAGE_URL=https://channels.weixin.qq.com/platform/
FINDER_BROWSER_HEADLESS=false
FINDER_PROFILE_DIR=./data/chromium-profile
FINDER_REQUEST_TIMEOUT=25
FINDER_STARTUP_TIMEOUT=45
```

必须替换 `FINDER_API_KEY`。该值用于机器人或调用程序访问接口。

本地扫码需要：

```dotenv
FINDER_BROWSER_HEADLESS=false
```

不要把 `.env`、API Key、Cookie、二维码或浏览器 profile 上传到 Git。

## 启动 Finder 网关

```powershell
powershell -ExecutionPolicy Bypass -File .\start_local.ps1
```

默认监听：

```text
http://127.0.0.1:8790
```

脚本会打开一个可见 Chromium。首次运行时：

1. 扫码登录视频号管理后台。
2. 进入内容管理页面。
3. 打开任意作品的分享菜单。
4. 等待网关捕获并定位短链模块。

不要关闭 Chromium。关闭运行网关的 PowerShell 窗口也会停止服务。

## 检查状态

另开一个 PowerShell：

```powershell
curl.exe http://127.0.0.1:8790/health
```

未登录或模块未加载时可能返回：

```json
{
  "data": {
    "browser_started": true,
    "ready": false,
    "reason": "short_link_service_not_found"
  }
}
```

完成登录并打开分享菜单后，应返回：

```json
{
  "data": {
    "browser_started": true,
    "ready": true,
    "reason": ""
  }
}
```

若页面已经登录但仍未就绪：

1. 确认当前页面是视频号管理后台。
2. 进入内容管理。
3. 关闭并重新打开作品分享菜单。
4. 使用带 API Key 的 reload 接口重新加载页面。

```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:8790/api/v1/finder/reload" `
  -Headers @{"X-API-Key" = "YOUR_API_KEY"}
```

## 生成短链

```powershell
$body = @{
  object_id = "OBJECT_ID"
  object_nonce_id = "OBJECT_NONCE_ID"
  scene = 40
} | ConvertTo-Json

Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:8790/api/v1/finder/share-url" `
  -Headers @{"X-API-Key" = "YOUR_API_KEY"} `
  -ContentType "application/json" `
  -Body $body
```

接口参数必须来自同一张视频号卡片的原始 XML。详见
[API调用说明](API调用说明.md)。

## 自定义监听地址或端口

默认只允许本机访问。如需让同一局域网内的其他设备访问：

```powershell
.\start_local.ps1 -HostAddress 0.0.0.0 -Port 8790
```

这会扩大网络暴露范围。必须同时配置 Windows 防火墙来源限制，并使用强
API Key。

仅修改端口：

```powershell
.\start_local.ps1 -Port 8791
```

## 停止、重启和登录状态

停止：

```text
在网关 PowerShell 窗口按 Ctrl+C
```

重启：

```powershell
.\start_local.ps1
```

登录数据保存在：

```text
data/chromium-profile
```

正常重启会继续使用该 profile。登录过期时，重新扫码即可。

不要在 Chromium 仍运行时同时启动第二个 Finder 网关实例，否则同一个
profile 可能被锁定。

## 更新代码

先停止网关，再执行：

```powershell
git pull
.\setup_local.ps1
.\start_local.ps1
```

重新执行安装脚本会更新 Python 依赖，但不会覆盖已有 `.env`。

## 可选元宝解析器

Finder 网关只负责生成 `sph`，不使用元宝。

如需把 `sph` 继续解析为视频信息，在 `.env` 中配置：

```dotenv
WXSPH_API_KEYS=change_me_to_a_long_random_key
WXSPH_COOKIE=
WXSPH_TIMEOUT=18
WXSPH_HOST_PORT=8787
```

启动：

```powershell
.\start_wxsph_local.ps1
```

健康检查：

```powershell
curl.exe http://127.0.0.1:8787/health
```

该组件依赖元宝会话，Cookie 会过期。它与 Finder 短链生成完全独立。

## 可选 Docker/VPS 部署

只有远程机器人无法访问本机时，才需要远程部署。

仓库保留 Docker Compose：

```bash
cp .env.example .env
chmod 600 .env
chmod +x deploy.sh test_gateway.sh finder_gateway/entrypoint.sh
./deploy.sh
```

Docker 版本通过 noVNC 完成首次扫码：

```bash
ssh -N -L 6080:127.0.0.1:6080 root@YOUR_SERVER
```

然后在本地浏览器打开：

```text
http://127.0.0.1:6080/vnc.html
```

公网部署只建议反向代理：

```text
POST /api/v1/finder/share-url
```

不要公开：

```text
noVNC 6080
/ready
/api/v1/finder/reload
```

Nginx 示例位于：

```text
nginx/sph.conf.example
```

将示例域名 `sph.example.com` 替换为自己的域名，并自行配置 HTTPS、限流和
访问控制。

## 常见问题

### Chromium 没有打开

检查：

```powershell
.\.venv\Scripts\python.exe -m playwright install chromium
```

并确认 `.env` 中：

```dotenv
FINDER_BROWSER_HEADLESS=false
```

### 端口已被占用

使用其他端口：

```powershell
.\start_local.ps1 -Port 8791
```

### profile 被锁定

关闭所有由该项目启动的 Chromium 和 Finder 网关进程，再重新启动。不要删除
profile，除非确认它已经损坏并接受重新扫码登录。

### 返回 401

请求头中的 `X-API-Key` 与 `.env` 中的 `FINDER_API_KEY` 不一致。

### 返回 503

Chromium 未启动、管理后台未登录，或分享菜单对应的短链模块尚未加载。

### 返回 502

卡片参数不匹配、页面接口发生变化，或管理后台没有返回有效 `sph`。
