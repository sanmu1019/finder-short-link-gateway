# 开发与测试说明

## 本地环境

建议：

```text
Windows 10/11
Python 3.12+
Git
Docker（仅验证可选容器部署时需要）
```

## 安装开发环境

先安装运行依赖和 Chromium：

```powershell
.\setup_local.ps1
```

再安装测试依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install pytest httpx
```

测试使用 FakeGateway，不会启动真实 Chromium，也不会请求视频号接口。

## 运行测试

```powershell
Push-Location finder_gateway
..\.venv\Scripts\python.exe -m pytest -q
Pop-Location
```

语法检查：

```powershell
.\.venv\Scripts\python.exe -m py_compile `
  finder_gateway\app\main.py `
  finder_gateway\tests\test_api.py `
  wxsph_api\wxsph_api.py
```

## 手工启动

通常使用：

```powershell
.\start_local.ps1
```

需要绕过脚本时，可直接运行：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app `
  --app-dir .\finder_gateway `
  --host 127.0.0.1 `
  --port 8790 `
  --env-file .\.env `
  --no-access-log
```

## 测试覆盖

当前测试包括：

- 正常转发 `objectId + objectNonceId`。
- 响应不回显 nonce。
- API Key 必填。
- 管理后台未就绪时返回 503。

涉及页面定位逻辑的改动还需要在已授权账号中进行手工集成测试。

## 修改 webpack 定位逻辑

管理后台前端更新后可能出现：

```text
webpack_runtime_not_loaded
webpack_require_not_captured
short_link_service_not_found
```

本地排查顺序：

1. 查看桌面 Chromium，确认页面已登录。
2. 进入内容管理页面。
3. 打开分享菜单以加载异步 chunk。
4. 调用 reload。
5. 检查 webpack runtime 名称。
6. 检查 `getObjectShortLink` 方法是否改名。

Docker/VPS 模式才需要通过 noVNC 查看 Chromium。

不要在 issue 中提供登录 Cookie、二维码、完整请求头或真实 nonce。

## API 兼容性

Finder bridge 同时保留 GET 和 POST：

- POST 用于新客户端和远程调用。
- GET 用于兼容已有机器人。

新增客户端应使用 POST。

## 日志要求

允许记录：

```text
objectId
nonce 长度
scene
状态码
耗时
```

禁止记录：

```text
完整 objectNonceId
Cookie
Authorization
二维码
浏览器 profile
```

## Docker 验证

只有修改 Docker 相关文件时才需要：

```bash
cp .env.example .env
docker compose config
docker compose build finder-gateway
```

验证后删除测试 `.env`，不要提交。

## 提交前检查

```powershell
git status --short
git diff --check
git check-ignore -v .env
git check-ignore -v data\chromium-profile
```
